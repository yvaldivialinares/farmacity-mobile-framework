## Feature: Email Code Request Form

### Description
Allows the user to enter their registered email address and request an access code to be sent by email. The submit button is disabled until the email field contains a non-empty value. On submission with a valid email, the user proceeds to a code verification screen.

### Usage
The user taps the email input and types their email address. The "Recibir código por email" button starts disabled (`enabled: false`) and becomes enabled once the email field has a value. Tapping the enabled button triggers sending an access code to the provided email and navigates to the next screen. If the email format is invalid or the email is not registered, an error may be shown depending on app validation behavior.

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 | Priority 3 |
|---------|------|------------|------------|------------|
| Email input | Input | `accessibility id: Correo electrónico` | `id: email-for-code` | `xpath: //android.widget.EditText[@resource-id="email-for-code"]` |
| Submit button | Button | `accessibility id: Recibir código por email` | `id: submit-button` | `xpath: //android.widget.Button[@resource-id="submit-button"]` |

> **Submit button observed state:** `enabled: false` when email field is empty. Becomes `enabled: true` when a value is entered.
> **Email input** has all three stable locator types — highest priority locator set in this app so far.

### Expected States/Validations
- Success: Valid registered email entered → tap submit → navigates to code-verification-screen
- Failure (empty field): Submit button remains disabled
- Failure (invalid format): App may show inline validation error (locator TBC — trigger manually to confirm)
- Failure (unregistered email): Error response from backend expected (locator TBC)
