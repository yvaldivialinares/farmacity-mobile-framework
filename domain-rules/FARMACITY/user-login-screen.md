## Feature: Welcome Header

### Description
Displays the screen title "¡Bienvenido!" and the instruction subtitle "Ingresá con tu correo y tu contraseña". These elements confirm the user has reached the credential login screen and orient them toward the form below.

### Usage
Both text elements are static and always visible on this screen. They do not respond to user interaction and do not change state. They serve as screen identification anchors for automation.

### Elements & Locators

| Element | Type | Priority 1 |

| Welcome title | Text | `xpath: //android.widget.TextView[@text="¡Bienvenido!"]` # FRAGILE: text-based, breaks on copy/localization changes |

> No resource-id or content-desc available on these elements — text-based XPath is the only option.

### Expected States/Validations
- Success: Both text elements visible immediately on screen load
- Failure: Missing or truncated text indicates the wrong screen was reached

## Feature: Credential Form

### Description
Allows the user to authenticate with their email address and password. The login button is disabled by default and only becomes enabled once both fields contain non-empty values. A password visibility toggle is available on the password field.

### Usage
The user taps the email input, types their email address, then taps the password input and types their password. The login button becomes enabled only after both fields are non-empty. Tapping the "Show password" toggle reveals or hides the password characters. When the login button is tapped with valid credentials, the user is navigated to the home-screen. With invalid credentials, an error is displayed and the user remains on this screen.

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 |
|---------|------|------------|------------|
| Email input | Input | `id: correoElectronico-input` | `xpath: //android.widget.EditText[@resource-id="correoElectronico-input"]` |
| Password input | Input | `id: password-input` | `xpath: //android.widget.EditText[@resource-id="password-input"]` |
| Show password toggle | Button | `accessibility id: Show password` | `xpath: //android.widget.Button[@content-desc="Show password"]` |
| Login button | Button | `accessibility id: Iniciar sesión` | `xpath: //android.widget.Button[@content-desc="Iniciar sesión"]` |

> **Login button observed state:** `enabled: false` when both fields are empty. Becomes `enabled: true` when both email and password fields contain non-empty values.

### Expected States/Validations
- Success: Login button enabled after both fields filled → tap → navigates to home-screen
- Failure (invalid credentials): Error message displayed; user remains on user-login-screen
- Failure (empty fields): Login button remains disabled (`enabled="false"`)
- Edge: Filling only email leaves login button disabled; password alone also leaves it disabled
- Toggle: Password field masked by default; "Show password" tap reveals plain text; second tap re-masks

## Feature: Account Navigation

### Description
Provides navigation links for users who need to recover their password or create a new account. These are secondary actions available below the main login form.

### Usage
Tapping "¿Olvidaste tu contraseña?" navigates to the password-recovery-screen. Tapping "Registrarme" navigates to the registration-screen. Both buttons are always enabled and visible regardless of the state of the credential form.

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 |
|---------|------|------------|------------|
| Password recovery button | Button | `accessibility id: ¿Olvidaste tu contraseña?` | `xpath: //android.widget.Button[@content-desc="¿Olvidaste tu contraseña?"]` |
| Register button | Button | `accessibility id: Registrarme` | `xpath: //android.widget.Button[@content-desc="Registrarme"]` |

### Expected States/Validations
- Success (recovery): Tap "¿Olvidaste tu contraseña?" → navigates to password-recovery-screen
- Success (register): Tap "Registrarme" → navigates to registration-screen
- Failure: Button tap does not trigger navigation
