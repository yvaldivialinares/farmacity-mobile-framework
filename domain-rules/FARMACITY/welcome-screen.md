## Feature: Welcome Greeting

### Description
Displays the greeting text "¡Hola!" when the app is first opened. This is a purely presentational element that confirms the app has loaded successfully and sets the tone for the onboarding flow.

### Usage
The greeting text is rendered immediately when the welcome screen loads. It is always visible and does not respond to user interactions. It does not change state based on any user action or system condition.

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 |
|---------|------|------------|------------|
| Greeting text | Text | `xpath: //android.widget.TextView[@text="¡Hola!"]` # FRAGILE: text-based, breaks on copy/localization changes | `—` |

> Priority 1 is text-based XPath only — no resource-id or content-desc available on this element.

### Expected States/Validations
- Success: TextView with text "¡Hola!" is visible on screen
- Failure: Text is missing, truncated, or displays incorrect content

## Feature: Authentication Navigation

### Description
Provides two distinct authentication paths from the welcome screen. Users choose between credential-based login ("Ingresar con usuario") or access-code-based login ("Ingresar con código"). Each button navigates to a different screen.

### Usage
Both buttons are always enabled and visible. No precondition is required to tap either button. Tapping "Ingresar con usuario" navigates to the user-login-screen. Tapping "Ingresar con código" navigates to the code-login-screen. There is no default selection or disabled state on this screen.

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 |
|---------|------|------------|------------|
| Login with user button | Button | `accessibility id: Ingresar con usuario` | `xpath: //android.widget.Button[@content-desc="Ingresar con usuario"]` |
| Login with code button | Button | `accessibility id: Ingresar con código` | `xpath: //android.widget.Button[@content-desc="Ingresar con código"]` |

### Expected States/Validations
- Success (user path): User is navigated to user-login-screen after tapping "Ingresar con usuario"
- Success (code path): User is navigated to code-login-screen after tapping "Ingresar con código"
- Failure: Button tap does not trigger navigation or app crashes
