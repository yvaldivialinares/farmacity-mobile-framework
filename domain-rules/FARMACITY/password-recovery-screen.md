## Feature: Password Recovery Form

### Description
Allows the user to enter the email address associated with their Farmacity account to request a password reset link. The submit button ("Recuperar contraseña") is disabled by default and becomes enabled once the email field contains a value.

### Usage
The user taps the email input and types their registered email address. The "Recuperar contraseña" button starts disabled (`enabled: false`) and becomes enabled once any value is entered in the field. On tap with a valid registered email, the app sends a recovery link and likely navigates to a confirmation screen. Email input placeholder is "ejemplo@hotmail.com".

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 | Priority 3 |
|---------|------|------------|------------|------------|
| Email input | Input | `accessibility id: Correo electrónico` | `id: email-input` | `xpath: //android.widget.EditText[@resource-id="email-input"]` |
| Recover password button | Button | `accessibility id: Recuperar contraseña` | `xpath: //android.widget.Button[@content-desc="Recuperar contraseña"]` | — |

> **Recover password button:** No `resource-id` observed — different from other submit buttons in this app (`submit-button`). Only `content-desc` is available as a stable locator.
> **Submit button state:** `enabled: false` on screen load. Becomes `enabled: true` when email field is non-empty.
> **Email input resource-id** (`email-input`) is distinct from `email-for-code` used on `code-login-screen` — no conflict across screens.

### Expected States/Validations
- Success: Valid registered email entered → tap "Recuperar contraseña" → recovery link sent → confirmation feedback shown
- Failure (empty field): Submit button remains disabled
- Failure (unregistered email): Error response from backend expected (locator TBC — requires unregistered email to trigger)
- Failure (invalid format): App may show inline validation (locator TBC)
