## Feature: Access Code Verification

### Description
Allows the user to enter the 6-digit access code received by email. A "resend code" link is available if the code was not received. The code field is pre-filled with a placeholder value ("000000"). This feature is part of the combined registration + verification flow.

### Usage
The user taps the code input field and enters the code received by email. The "¿No recibiste el código? Solicitar otro código" link triggers a new code to be sent. The submit button ("Continuar") becomes enabled only when both the code field AND all required profile fields are filled. Code field placeholder text is "000000".

### Elements & Locators

| Element | Type | Priority 1 | Priority 2 | Priority 3 |
|---------|------|------------|------------|------------|
| Code input | Input | `accessibility id: Ingresá el código de acceso` | `id: codigo-input` | `xpath: //android.widget.EditText[@resource-id="codigo-input"]` |
| Resend code link | Text/Link | `xpath: //android.widget.TextView[@text="¿No recibiste el código? Solicitar otro código"]` # FRAGILE: text-based · `clickable: false` observed — may require tap on parent ViewGroup | — | — |

> **Code input placeholder:** "000000" — clear before typing.
> **Resend link:** observed as `clickable: false` TextView. In React Native, this often means the tap is handled by a parent element. If direct tap fails, use parent ViewGroup locator or coordinate-based tap.

### Expected States/Validations
- Success: Enter valid code → (combined with profile form) → "Continuar" becomes enabled
- Failure: Wrong code → error message expected (locator TBC — requires real code to trigger)
- Edge: Tap resend link → app sends a new code → instruction text may update email display

## Feature: Profile Creation Form

### Description
A registration form requiring the user to provide personal information to create their Farmacity account. Displayed on the same screen as the code verification field. Mandatory fields are: Nombre, Apellido, Género, Fecha de nacimiento. Número de documento has no `*` marker and appears optional.

### Usage
The user fills in each form field. Text inputs accept free text with placeholder text as hints ("Tu nombre", "Tu apellido", "Numero de documento"). Género opens a selection dialog/dropdown. Fecha de nacimiento opens a date picker. The "Continuar" button remains disabled until all mandatory fields (code + required profile fields) are completed.
